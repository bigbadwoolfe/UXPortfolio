A company on a journey to simplify playing the lottery in groups.

The OLG announced their own version of this application in 2013 and launched it in 2015. This validated the need for such a product, but at the same time it rendered this one obsolete. The code for LotteryPals.com never left the development servers.

The Service

The people behind LotteryPals.com created it to solve their own problem, and the problem of many - finding people to play the lottery in groups.

LotteryPals.com would allow users to buy and track their lottery tickets online, and would simplify the process of creating, managing and joining player groups.

My Role

I led the design of the application from an idea to a functional product.

I documented user requirements, laid out the information architecture and content strategy, created mockups, prototypes and wire frames, designed the graphics and interaction and participated in the actual implementation of the product.

User Research

I communicated with target users to identify pain points and created a list of features to address the same.

Planning and Project Management

I created a project road map, from user profiling through architecture and design to implementation. I prioritized and balanced features based on user needs and business goals.

Information Architecture, UX Design & Validation

I produced use case and workflow diagrams, prototyped, mocked up and wireframed the application, created graphics and collected and analyzed feedback.

UI Implementation

I partially wrote the UI with mock data in HTML with CSS and JS, under ASP.NET MVC. I tested for responsiveness across multiple screen sizes and cross browser compatibility.

Discovery

The problem at hand had two major sides to consider - how lottery players are organizing themselves into groups and what was the legally correct way to do it. These two were not always the same.

I learned that lottery players who manage groups that play together have the following pains:

  • Collecting money
  • Scanning tickets and distributing copies
  • Skipping draws
  • Distributing small winnings, free plays, etc

The group members and the ones wanting to join a group had a different set of problems:

  • Missing payments (due to time off work for example)
  • Difficulty to opt out occasionally
  • Finding groups to join

In addition, the OLG imposes strict rules for playing in groups that a company would be required to follow.

Finally, the company needed modern web presence how they would sign up new players. Here the goal is to explain the business model in simple terms while gaining the customer's trust.

Definition

This was a heavily data-driven application. User types with different privileges had to exist and the number of use cases per user type was staggering. To keep track of who does what, I had to create a User-Privilege matrix. The creation of site maps and workflow diagrams heavily depended on this matrix.

The two major components would be the public website and the member's portal, which would then offer different functionality to the users according to the use case diagrams.

Design

Due to the nature of the application, it was imperative that users would be left with both a sense of security and playfulness from the very beginning. This project was happening right around the time when all lottery and gambling websites in the US were getting shut down, so creating a user experience that instills confidence was of utmost importance.

Gaining the trust of a customer is infinitely easier than gaining it back.

Influencing feelings, changing perceptions

Research has shown that users perceive a well-built, precise and accurate UI as more secure. So the goal was to deliver exactly that - a professionally designed and built application with attention to detail paid to every pixel, every click, every bit of information on the screen. There could not be any room for doubt.

For added sense of security, I went above and beyond to remove possibilities for mistakes from all workflows. Every user action was to be confirmed unless the user had opted out, and every action was to be reversible directly by the user.

To achieve playfulness, use of vibrant, but not overwhelming colors and relevant graphics was made.

Simplicity

The value proposition for target users was the alleviated pain that comes with playing the lottery in groups and the ability to buy tickets from the comfort of their homes. LotteryPals.com could not be the source any new pain.

To improve the user experience for users of all technical skill levels and across a variety of devices, each workflow and each entry form was designed for simplicity and error prevention. Depending on the context, visual elements like sliders, calendar widgets and toggle switches were used to replace traditional data entry fields. Additionally, extensive field validation was created to allow for early error detection.

Challenges

A major design challenge was maintaining balance between a UI that instills confidence and feeling of security (the norm being very rigid online banking applications) and one that is playful.

To overcome, a simple color scheme utilizing contrasting colors was created and color was used to highlight web elements in a consistent manner, while keeping most of the content on a simple plain background. While also decorative, colors were used primarily with a function to make important application components more noticeable.

Lessons Learned

Don't guess what the users want

... ask them!

User interviews are invaluable tool. They provide validation to some ideas and completely shatter others. It is those users that will drive the business however, so when an idea is shattered, pick up the pieces and make it better.

Stop planning and start doing

No amount of pre-planning will yield a perfect solution the first time. Refining the product through iteration will happen no matter what, so the sooner it starts, the better. Get a good understanding of what is to be built, and start building it.

Reflection

As some of the best products out there, LotteryPals.com was created out of a real need to simplify a task that most people do. Even though the project got shut down before it ever launched, proper user centered design methodologies were used during the design what resulted in a web application that excelled at the tasks it set out to accomplish.

It was a great journey and an awesome learning experience, one that has helped me many times since.

A company on a journey to simplify playing the lottery in groups.

The Ontario Lottery and Gaming Corporation implemented their own version of this application validating the need, but at the same time rendering this one obsolete. Code never left the development servers.

The Service

LotteryPals.com would allow users to buy and track their lottery tickets online, and more importantly, would simplify the process of creating, managing and joining groups.

My Role

I led the design of the application from an idea to a functional product.

I documented user requirements, created use cases and information architecture, designed workflows, graphics, interaction and participated in the actual implementation of the product.

User Research

I communicated with target users to identify pain points and created a list of features to address the same.

Planning and Project Management

I created a project road map, from user profiling through architecture and design to implementation. I prioritized and balanced features based on user needs and business goals.

Information Architecture, UX Design & Validation

I produced workflow diagrams, designed layouts and UI elements and collected target user feedback.

UI Implementation

I wrote the UI with mock data in HTML with CSS and JS, under ASP.NET MVC. I tested for responsiveness across multiple screen sizes and cross browser compatibility.

Discovery

The problem at hand had two major sides to consider - how lottery players are organizing themselves into groups and what was the legally correct way to do it. These two were not always the same.

I learned that lottery players who manage groups that play together have the following pains:

  • Collecting money
  • Scanning tickets and distributing copies
  • Skipping draws
  • Distributing small winnings, free plays, etc

The group members and the ones wanting to join a group had a different set of problems:

  • Missing payments (due to time off work for example)
  • Difficulty to opt out occasionally
  • Finding groups to join

In addition, the Ontario Lottery and Gaming Corp imposes strict rules for playing in groups that a company would be required to follow.

Finally, the company needed modern web presence how they would sign up new players. Here the goal is to explain the business model in simple terms while gaining the customer's trust.

With all this information I could create profiles for each identified user type (public, regular players, group managers, system administrators) and feature sets required by each user type and get a rough idea on how to present this in a self-explanatory way.

Definition

The application is a heavily data-driven one. Different user types would exist, each with different capabilities. For that, I had to create a user-privilege matrix that I used to create the site maps and user workflows.

The two major components would be the public part of the web application and the member's portal.

Design

Due to the nature of the application, it was imperative that users would be left with both a sense of security and playfulness from the very beginning. This project was happening right around the time when all lottery and gambling websites in the US were getting shut down, so of utmost importance was to create a user experience that would instill confidence in the players.

Gaining the trust of the customer is infinitely easier than regaining it.

Influencing feelings, changing perceptions

Research has shown that users perceive a well-built, precise and accurate UI as more secure. So the goal is to deliver exactly that - a professionally designed and built application with attention to detail paid to every pixel, every click, every bit of information on the screen. There could not be any room for doubt.

For added sense of security, I went above and beyond to remove possibilities for mistakes from all workflows. Every action was to be confirmed unless the user requested not to, and every action was to be reversible directly by the user.

To achieve playfulness I used a vibrant color scheme and relevant graphics.

Simplicity

The value proposition for target users was the alleviated pain that comes with playing the lottery in groups and the ability to buy tickets from the comfort of their homes. LotteryPals.com could not be the source any new pain.

To improve the user experience for users of all technical skill levels and across a variety of devices, each workflow and each entry form was designed for simplicity and error prevention. Depending on the context, visual elements like sliders, calendar widgets and toggle switches were used to replace traditional data entry fields. Additionally, extensive field validation was created to allow for early error detection.

Challenges

A major design challenge was maintaining balance between a UI that instills confidence and security (the norm being very rigid online banking applications) and one that suggests playfulness.

To overcome, a simple 3-color scheme was created and colors were used to highlight web elements, while keeping most of the content on a simple white background.

Lessons Learned

Don't guess what the users want

... ask them!

User interviews are invaluable tool. They provide validation to some ideas and completely shatter others. It is those users that will drive the business however, so when an idea is shattered, pick up the pieces and make it better.

Stop planning and start doing

No amount of pre-planning will yield a perfect solution the first time. Refining the product through iteration will happen no matter what, so the sooner it starts, the better. Get a good understanding of what is to be built, and start building it.

Reflection

As some of the best products out there, LotteryPals.com was created out of a real need to simplify a task that most people do. Even though the project got shut down before it ever launched, proper user centered design methodologies were used during the design what resulted in a web application that excelled at the tasks it set out to accomplish.

It was a great journey and an awesome learning experience, one that has helped me many times since.
